updSH — Shell Reference

A friendly Linux shell with a built-in package manager, customizable prompt, alias system, and job control.

Built-in Commands

cd <dir>builtin
Change the current working directory. Defaults to ~ if no argument given.
exit [code]builtin
Exit the shell with an optional exit code (default 0).
pwdbuiltin
Print the current working directory.
echo [text...]builtin
Print text to stdout. Joins all arguments with spaces.
clearbuiltin
Clear the terminal screen.
export KEY=VALUE [...]builtin
Set environment variables. Multiple KEY=VALUE pairs accepted.
historybuiltin
Show command history with line numbers.
helpbuiltin
Open this help page in your browser.
jobsbuiltin
List all background jobs with their IDs and status.
fg [id]builtin
Bring a background job to the foreground. Defaults to most recent job.
bg [id]builtin
Resume a stopped job in the background. Defaults to most recent job.
source <file>builtin
Read and execute commands from a file.
type <cmd>builtin
Show whether a command is a builtin, external, or not found.
alias [name=value ...]builtin
Define an alias or list all aliases. Use alias name=value to set.
unalias <name>builtin
Remove an alias.

Package Manager: pkg

Manage packages from the Vercel-hosted registry or built-in collection.

pkg install <package> [packages...]
Install one or more packages. Downloads source, compiles C/Rust, or builds from git repos automatically.
pkg remove <package> [packages...]
Remove an installed package.
pkg list
List all currently installed packages.
pkg search [query]
Search available packages by name or description. Omitting query shows all.
pkg info <package>
Show detailed information about a package (version, description, type, author, dependencies).
pkg update
Update all installed packages to the latest version.
pkg help
Show package manager help text.

Package Types

  • source — Shell alias scripts sourced on shell startup.
  • compile — C or Rust source code compiled with gcc or rustc into a binary.
  • build — Clones a git repository, runs cmake, cargo, or make to build real upstream tools (fastfetch, ripgrep, fd, bat, fzf, etc.).

Shell Features

Auto-completion (Tab)

Press Tab to complete commands (first word) or file paths. Press Tab again to cycle through matches.

Fish-style autosuggestions appear in gray as you type based on command history. Press Ctrl+F or Right Arrow to accept.

Pipes

Chain commands with |: cmd1 | cmd2 | cmd3

Redirections

  • > file — Redirect stdout to file (overwrite)
  • >> file — Append stdout to file
  • < file — Read stdin from file
  • 2> file — Redirect stderr to file
  • 2>&1 — Redirect stderr to stdout

Background Jobs

Append & to run a command in the background: cmd &

Use jobs to list, fg to bring to foreground, bg to resume.

Command Sequencing

Separate multiple commands with ;: cmd1; cmd2; cmd3

Variables

  • $VAR — Expand variable
  • ${VAR} — Variable with explicit boundaries
  • ~ — Expand to home directory

Command History

  • Up/Down arrows — Navigate history
  • Persistence — History saved to ~/.local/share/updsh/history.txt

Prompt Customization

Configure in ~/.config/updsh/env or via environment variables:

UPD_PROMPT_STYLE=multiline     # multiline, powerline, minimal, plain
UPD_COLOR_USER=green
UPD_COLOR_HOST=blue
UPD_COLOR_PATH=yellow
UPD_COLOR_GIT=red
UPD_COLOR_EXIT=red
UPD_SHOW_GIT=yes               # Show git branch in prompt
UPD_SHOW_EXIT_CODE=yes         # Show exit code on failure

Prompt styles:

  • multiline — Two-line themed prompt with decorative border
  • powerline — Powerline-style colored background segments
  • minimal — Compact single line with git branch
  • plain — Simple ~/path $ prompt

Security

All packages are automatically scanned for malicious patterns:

  • Destructive commands (rm -rf /, mkfs, dd if=/dev/zero)
  • Fork bombs and pipe-to-shell patterns
  • sudo rm -rf, raw device writes (/dev/sda)
  • Remote code execution via eval $(curl

Malicious packages are deleted immediately.
